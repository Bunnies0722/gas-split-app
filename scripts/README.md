# ガソリン価格 自動取得スクリプト

資源エネルギー庁「石油製品価格調査」の週次データ（レギュラー/ハイオク/軽油の全国平均価格、および北海道/東北/関東/中部/近畿/中国/四国/九州/沖縄の9地域別価格）を取得し、`prototype/price.json` に保存します。

地域別価格は、出発地点・到着地点の都道府県から算出する「2地点の平均単価」提案機能（`prototype/index.html`側）で使われます。

## 使い方

```bash
cd scripts
npm install
npm run fetch:debug   # まずこれで実際のExcelの中身を確認
npm run fetch          # price.json を生成
```

## 重要な注意

- **ブラウザのJavaScriptから直接資源エネルギー庁のサイトを呼び出すことはできません。** CORS（Cross-Origin Resource Sharing）というブラウザ側の制約により、許可されていない外部サイトへの直接fetchはブロックされます。資源エネルギー庁のサイトは一般的な情報公開サイトであり、API向けのCORS許可は出していません。
  - そのため、このスクリプトは**Node.js（サーバー側）で実行**します。サーバー同士の通信にはCORSの制約がないため問題なく取得できます。
  - 将来、Webアプリ化するときは「このスクリプトを定期実行（例：Vercel Cronで週次）→結果をDBや`price.json`に保存→フロントエンドは自分のサーバーのその結果を読むだけ」という構成にします（開発計画.mdのPhase 3参照）。

- **Excelのレイアウトは未検証です。** 私（Claude）の実行環境からはenecho.meti.go.jpへの通信がネットワーク制限でブロックされており、実際のファイルを直接確認できていません。`findFuelPrices`関数は一般的な統計表のレイアウト（油種名の見出し行→「全国」の行）を想定して書いていますが、実際の見出し表記やレイアウトが異なる場合は動作しません。

  初回は必ず `npm run fetch:debug` を実行し、表示されるシートの内容を見ながら `fetch-gas-price.js` 内の `FUEL_KEYWORDS` や `findFuelPrices` のロジックを実際のファイルに合わせて調整してください。

- パース処理自体のロジック（ヘッダー行検出→該当行から値を取り出す部分）は、実際に取得したExcelの構造（経済産業局別シート）に基づいて検証済みです。

## price.json の形式

```json
{
  "fetchedAt": "2026-06-29T08:56:40.205Z",
  "weekOf": "2026-06-22",
  "sourceUrl": "https://www.enecho.meti.go.jp/.../260624.xlsx",
  "unit": "円/L",
  "prices": { "regular": 170, "highoctane": 181, "diesel": 159 },
  "regionPrices": {
    "regular": { "北海道": 171, "東北": 168, "関東": 169, "中部": 167, "近畿": 169, "中国": 170, "四国": 170, "九州": 173, "沖縄": 178 },
    "highoctane": { "...": "(同様の9地域)" },
    "diesel": { "...": "(同様の9地域)" }
  }
}
```

`prices` は全国平均（地域が特定できない場合のフォールバック用）、`regionPrices` は9地域それぞれの平均価格です。
